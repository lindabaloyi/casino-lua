# Plan to Add Game Menu with 2-Player and 3-Player Options

## Overview
The current game starts directly on the board with a hardcoded 2-player setup (Player vs Dealer). To add a simple menu screen with options for 2-player and 3-player modes, we need to modify the game flow to start on a menu, allow player count selection, and update the game logic to support variable player counts with turn-based play.

## Current State Analysis
- Game uses ScreenManager with "home" (unused) and "game" screens.
- main.lua switches directly to "game".
- GameState hardcodes 2 players: "Player" and "Dealer".
- GameBoard renders capture piles for players[1] and players[2].
- No turn-switching logic; currentPlayerIndex is fixed at 0.
- Actions are executed with playerIndex=0 only.

## Proposed Changes

### 1. Update Game Entry Point
- Modify `main.lua` to switch to "home" screen instead of "game".

### 2. Enhance HomeScreen for Player Selection
- Replace single "PLAY" button with two buttons: "2 Players" and "3 Players".
- On button click, switch to "game" screen and pass the selected player count.

### 3. Update ScreenManager
- Modify `switch` method to accept and pass additional arguments (player count) to screen:load().

### 4. Modify GameState for Variable Players
- Update `GameState:new(numPlayers)` to initialize the correct number of players.
- Update `start()` to deal hands appropriately (e.g., 10 cards each, adjust deck if needed for 3 players).
- Ensure players array is dynamic.

### 5. Update GameBoard for Multi-Player Support
- Modify rendering to display capture piles for all players (position them accordingly).
- Update currentPlayerIndex handling to cycle through all players.

### 6. Implement Turn-Based Logic
- Add turn-switching after successful actions in InputHandler.
- Update currentPlayerIndex to next player (modulo numPlayers).
- Ensure UI highlights the current player's turn.

### 7. Adjust Game Logic for Turns
- Modify actions to use currentPlayerIndex instead of hardcoded 0.
- Ensure hand management for multiple human players (may need to display different hands).

### 8. UI Adjustments
- Reposition capture piles for 3 players.
- Update any player-specific rendering.

## Assumptions
- All players are human; no AI for dealer/computer players.
- 3-player mode means 3 human players taking turns.
- Card dealing: 10 cards per player for both modes (adjust if deck runs short).
- Game ends when hands are empty, as per standard Cassino rules.

## Potential Challenges
- Hand management: Currently only shows one hand (playerHand). Need to show current player's hand.
- Input handling: Ensure only current player can make moves.
- Game flow: Handle turn progression and win conditions for multiple players.

1. Update main.lua to start on "home".
2. Modify HomeScreen to add player selection buttons.
3. Update ScreenManager to pass player count.
4. Refactor GameState for dynamic player count.
5. Update GameBoard rendering and logic.
6. Implement turn switching in InputHandler after actions.
